1. Protocol : 통신을 하기 위한 규약
2. 소켓의 생성
   int socket(int domain, int type, int protocol);
   -> 성공 시 파일 디스크립터, 실패 시 -1 반환
   domain: 소켓이 사용할 프로토콜 체계 정보 전달
   type: 소켓의 데이터 잔송방식에 대한 정보 전달
   protocol: 두 컴퓨터간 통신에 사용되는 프로토콜 정보 전달

   2-1. 프로토콜 체계
      PF_INET ; IPv4 인터넷 프로토콜 체계
         2^32bit로 8bit 4자리로 주소를 나타낸다.
      PF_INET6; IPv6 인터넷 프로토콜 체계
         2^128bit로 16bit 8자리 16진수로 주소를 나타낸다.
      PF_LOCAL; 로컬 통신을 위한 UNIX 프로토콜 체계
      PF_PACKET; LOW LEVEL 소켓을 위한 프로토콜 체계
      PF_IPX; IPX 노벨 프로토콜 체계

   2-2. 소켓의 타입
      소켓의 데이터 전송방식을 의미
      프로토콜 체계가 정해져도 세부적인 전송방식이 나뉘어 진다.
      2-2-1. 연결지향형 소켓 (SOCK_STREAM)
         단순 연결로 데이터를 전송하는 방식
	 마치 화물 열차 
	 중간에 데이터 소멸되지 않고 목적지로 전송
	 전송 순서대로 데이터가 수신 (순차적으로 전달 된다. 섞이지 않는다.)
	 전송되는 데이터의 경계가 존재 하지 않음
	 (BUF에 데이터가 다 채워져야지만 데이터를 읽어 들인다.
	 이것은 마치 C/C++에서 파일스트림의 방식과 유사한것 같다.)
	 소켓의 연결은 1 대 1
	 "신뢰성 있는 순차적인 바이트 기반의 연결지향 데이터 전송 방식의 소켓"
      2-2-2. 비 연결지향형 소켓(SOCK_DGRAM)
         전송된 순서에 상관없이 가장 빠른 전송을 지향
	 전송된 데이터 손실,파손의 우려가 있음
	 전송되는 데이터의 경계가 존재
	 한 번에 전송 할 수 있는 데이터의 크기가 제한
	 마치 오토바이 택배
	 "신뢰성과 순차적 데이터 전송을 보장하지 않는, 고속의 데이터 전송을 목적으로 하는 소켓"
   
   2-3. 프로토콜 정보
      2-1, 2-2와 완전하게 동일한 전송방식이 있을 경우 해당 인자를 통해 좀 더 
      추가적인 정보를 전달할 수 있게 함
      IPv4의 연결지향형 데이터 전송 소켓의 경우 IPPROTO_TCP 라는 인자를 입력
      IPv4의 비 연결지향형 데이터 전송 소켓의 경우 IPPROTO_UDP라는 인자를 입력
